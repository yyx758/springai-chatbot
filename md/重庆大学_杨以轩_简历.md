## 项目名称: AI Studio 智能客服系统

## 项目背景

希望通过搭建属于自己的 AI 小平台来感受 AI 浪潮。本项目在 AI Agent 辅助下开发，在真实项目中学习如何使用 AI、如何用 AI 赋能。其余技术栈随需求迭代加入，最终形成包含双模型对话、RAG 检索增强、微服务架构、Docker 容器化部署、权限管理、全栈部署的完整产品。

**项目描述**: 基于 Spring Boot 3.2 + Spring Cloud 的微服务智能客服平台，接入 DeepSeek 大模型和本地 Ollama 模型，支持双模型运行时切换、流式对话（SSE）、多轮上下文记忆、多模态图文混合输入（图片自动路由到 llava 视觉模型）。采用 Kafka 事件驱动异步通信，Nacos 服务注册发现，Docker Compose 一键部署（7 个服务）。项目部署于阿里云轻量服务器，通过 Docker 容器化运行。

**技术栈**：Spring Boot、Spring Cloud Gateway、Spring AI、MyBatis-Plus、MySQL、Redis、Kafka、Nacos、JWT、Docker、Thymeleaf、Nginx、Cloudflare Tunnel、Ollama llava、PDFBox、Apache POI

## 个人职责:

### 1.Docker 容器化 + 一键部署

- 将单体应用拆分为 3 个微服务（chatbot-service / file-service / gateway）+ 4 个基础设施（MySQL / Redis / Kafka / Nacos），编写 Dockerfile 多阶段构建 + docker-compose 编排，通过环境变量注入实现配置与代码分离
- **数据成果**：开发/测试/生产环境一致，**一条命令 `docker compose up -d` 完成部署**；多阶段构建将镜像从 763MB（含 Maven）压缩至 **400MB**（仅 JRE）

---

### 2.Kafka 事件驱动 + 异步解耦

- 用Kafka Topic（chat.events / notification.events / knowledge.events）+ 独立 Consumer Group + 手动 ACK 解决同步处理导致 HTTP 响应阻塞、邮件发送失败影响主流程
- **数据成果**：邮件发送与注册解耦，失败不影响注册；聊天记录异步持久化，HTTP 响应延迟不受影响；KnowledgeEventConsumer 自动刷新 RAG 缓存

---

### 3.文件管理微服务 + 文档解析

- 独立 file-service 微服务，抽象 FileStorage 接口（LocalStorage / MinioStorage），集成 Apache PDFBox + POI 实现 PDF/DOCX/TXT 文档自动解析，上传即得文本内容
- **数据成果**：支持图片自动压缩 + 缩略图生成；知识库文档上传后自动解析，零手动输入；文件管理页面支持分页、下载、预览

---

### 4.Redis 缓存 + 异步写回

- 用**Redis List** 存储聊天历史 + `CompletableFuture` 异步写回解决了每次对话都查 MySQL，单次查询毫秒级，多轮对话反复查库
- **数据成果**：缓存命中后查询速度从毫秒级降至微秒级，降幅约 **1000 倍**（`System.nanoTime()` 实测对比），缓存命中率超过 90%

---

### 5.Semaphore 信号量 + Ollama 并发排队

- 用`Semaphore(1)` 信号量 + `tryAcquire(120s)` 超时 + SSE 推送排队状态解决Ollama 单线程推理，多人同时使用直接报 `Connection refused`
- **数据成果**：从"并发冲突直接报错"变为"**120 秒内排队依次处理，零报错**"，前端实时显示排队状态

---

### 6.Nginx 反向代理 + 限流

- 用Nginx `limit_req 10r/s` 限流 + `proxy_buffering off` 流式代理 + `server.address 127.0.0.1` 端口隐藏解决了后端 8080 端口直接暴露公网无防护；SSE 流式被 Nginx 默认缓冲打断
- **数据成果**：API 限制为每 IP 每秒 **10 次**；流式对话零缓冲延迟；后端端口不对公网暴露

---

### 7.JWT 双令牌 + Token 原子轮转

- 用**JWT**（HMAC-SHA256 签名）+ Redis `getAndDelete` 原子操作 + 前端 `refreshPromise` 互斥锁解决了单 Token 泄露后无法撤销；多个请求同时触发刷新导致 Token 竞争
- **数据成果**：Access Token **30 分钟**自动过期，泄露窗口缩至 30 分钟；刷新令牌可随时吊销；N 个 401 并发只触发 **1 次**刷新请求
  
---

### 8.自研 RAG 关键词评分算法

- 用三层评分模型（标题 40/正文 30/标签 20）+ 中文 2-gram 子词切分解决AI 对知识库内容一无所知，全靠预训练数据"猜测"
- **数据成果**：零向量数据库依赖，单文档评分**亚毫秒级**；AI 回复变为可溯源引用，检索失败自动降级

---

### 9.架构容错与降级

- 用`ObjectProvider.getIfAvailable()` + `@Async` 异步写 + Redis 故障自动 fallback MySQL解决单模型不可用时应用启动崩溃；聊天记录写入阻塞 HTTP 响应
- **数据成果**：单模型不可用时应用**正常启动**；记录写入异步非阻塞，HTTP 响应延迟不受影响

---

### 10.多模态图文混合输入

- 用 Spring AI `UserMessage.builder().text().media().build()` 构建多模态消息 + `OllamaOptions` 请求级模型覆盖 + file-service 独立管理图片存储解决了用户需要上传图片让 AI 分析（如识别场景、OCR、分析图表），纯文本对话无法处理视觉任务；Base64 存入 MySQL 导致存储膨胀 33%
- **数据成果**：支持 jpg/png/gif/webp 最大 **10MB** 图片上传、粘贴、拖拽三种方式；图片存储在 file-service，文件管理统一查看；自动路由到 llava 视觉模型，**零代码切换**
